Question 1

What is the difference between var and let keywords?

1. var is in JS from the beginning. let was introduced in ES2015/ES6.

2. let has a block scope. var has a function scope.

3. var gets hoisted, which the following code explains.


        let x = function() {
          if (true) {
            console.log(v); // undefined
            console.log(l); // l is not defined
            var v = 2;
            let l = 1;
            console.log(l); // 1
          }
           console.log(v); // 2
           console.log(l); // l is not defined
        }
        x();
Question 2

What is the difference between == and === signs?

1. == and === are both comparison operators.

2. == compares value only.

3. === compares value and type.

The following example explains the difference.


          '1' == 1 
    //true(both values convert to string type and compare;
    //and the String '1' are equiv to String '1')

          '1' === 1 
    // false (compare first types;
    // and the String type and Number type are not equiv)

          let l = 1;

Question 5

What is the use of Arrow function?

Arrow functions do not have their own this value. The value of this inside an arrow function is always inherited from the enclosing scope. Consider the following example:


          const profile = {
            firstName: "",
            lastName: "",
            setName: function (name) {
              let splitName = function (n) {
                let nameArray = n.split(" ");
                this.firstName = nameArray[0];
                this.lastName = nameArray[1];
              };
              splitName(name);
            },
          };
          
          profile.setName("john doe");
          console.log(profile.firstName);//"", nothing ))
          console.log(window.firstName);//"john"

This has happened because splitName function is not a constructor function and only sets firstName and lastName to a window object.

But, if we replace ordinary function with an arrow function, it will get this from the profile object.


          const profile = {
            firstName: "",
            lastName: "",
            setName: function (name) {
              let splitName = (n) => {
                let nameArray = n.split(" ");
                this.firstName = nameArray[0];
                this.lastName = nameArray[1];
              };
              splitName(name);
            },
          };
          
          profile.setName("john doe");
          console.log(profile.firstName);//"john"

Note, that arrow functions also do not have arguments array, and can not use as a constructor function.

Question 6

What is prototypal inheritance?

Every object has a property called a prototype. This property contains methods that the object inherited from the parent object. When you call some object`s method JS first search it from the own object`s properties and then search it in the methods from the prototype.

The example of prototype inheritance:


          let Car = function (model) {
            this.model = model;
          };
          Car.prototype.getModel = function () {
            return this.model;
          };
          
          let toyota = new Car("toyota");
          console.log(toyota.getModel()); //"toyota"
          
          let opel = new Car("opel");
          console.log(opel.getModel()); //"opel"

What is the difference between function declaration and function expression?

First of all function declaration and function expression have different syntax. Function declaration declares a named function when function expression assigns an anonymous function to a variable.

Function declaration is enabled in code between its declaration when function expression is not. Function expression is assigned to the variable, so it behaves like a variable and has a variable scope as well. To see this you can consider the following example:


          console.log(functionDeclaration());
          //"Function Declaration"
          console.log(functionExpression());

          //Uncought ReferenceError: 
          //functionExpression is not defined


          function functionDeclaration() {
            console.log('Function Declaration')
          };

          let functionExpression = function () {
            console.log('Function Expression')
          };

When you need to pass a function to another function, you can not use function declaration, but only a function expression.

Question 8

What is promises and why do we use it?

A promise is an object that contains a result of the async operation. It accepts resolve and reject functions. Resolve is the function that should be called when the promise executes successfully and reject is the function that should be called when the promise encounters an error. The syntax of the promises looks like this:


          let p = new Promise(function (resolve, reject) {
            let workDone = true; // some time consuming work
            if (workDone) {
              //invoke resolve function passed
          
              resolve('success promise completed')
            }
            else {
              reject('ERROR , work could not be completed')
            }
          })

The promises provide you a great opportunity to avoid so-called "callbacks hell" when you need to pass a callback function to be run after the execution of another function.

Question 9

What is setTimeout()?

setTimeout() is one of the many timing events. The window object allows the execution of code at specified time intervals. This object has provided setTimeout() to execute a function after a certain amount of time. It takes two parameters as arguments. One is the function and the other is the time that specifies the interval after which the function should be executed. The trick is when you set 0 milliseconds to delay the execution, the callback function anyway will be placed in the queue and executed after all sync functions from the stack.


          setTimeout(() => {
            console.log('a');//third
          }, 0);

          console.log('b');//first 
          
          console.log('c');//second
Question 10

What is closure and how do we use it?

A closure is the combination of a function and the lexical environment within which that function was declared. Example:


          let obj = function () {
            let i = 0;
            return {
              // setI(k) getI() are the inner functiona, closures
              setI(k) {
                i = k;
              },
              getI() {
                return i;
              },
            };
          };
          
          let x = obj();
          
          x.setI(2);
          console.log(x.getI()); //2
          //x is an Object
          //x.__proto__.[[Scopes]].Closures[0]={i:2}

Practical uses of the closures is emulating private methods. Languages such as Java provide the ability to declare methods private, meaning that they can only be called by other methods in the same class. JavaScript does not provide a native way of doing this, but it is possible to emulate private methods using closures. Example:


          const counter = (() => {
            let privateCounter = 0;
            let changeBy = (val) => (privateCounter += val);
            return {
              increment: () => changeBy(1),
              decrement: () => changeBy(-1),
              value: () => privateCounter,
            };
          })();

          console.log(counter.value()); //0
          counter.increment();
          counter.increment();
          console.log(counter.value()); //2
          counter.decrement();
          console.log(counter.value()); //1
